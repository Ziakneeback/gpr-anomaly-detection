# Практическая часть магистерской диссертации

Тема: **Разработка алгоритма и программного обеспечения по определению аномалий георадиолокационных сигналов**.

В проекте реализовано настольное приложение на Python с графическим интерфейсом. Программа позволяет загрузить CSV-файл с трассами георадиолокационного сигнала, выполнить предобработку, найти аномальные области и сформировать текстовый отчет.

## Запуск в VS Code

1. Откройте папку проекта в Visual Studio Code.
2. Убедитесь, что выбран интерпретатор Python 3.10 или новее.
3. Запустите файл:

```powershell
python gpr_anomaly_app.py
```

Дополнительные библиотеки не требуются. Используются только стандартные модули Python: `tkinter`, `csv`, `math`, `statistics`, `random`, `pathlib`.

## Формат входных данных

CSV-файл должен содержать числовую матрицу:

- строка соответствует одной трассе георадара;
- столбец соответствует временному отсчету или глубинному интервалу;
- значения являются амплитудами отраженного сигнала.

Пример:

```csv
0.12,0.15,0.11,0.03
0.10,0.13,0.09,-0.01
0.14,0.20,0.42,0.08
```

## Реализованный алгоритм

Алгоритм состоит из четырех этапов.

1. **Загрузка или генерация данных.** Пользователь может загрузить реальную радарограмму в CSV-формате или создать демонстрационный набор сигналов с искусственно добавленными аномальными зонами.
2. **Удаление фоновой составляющей.** Для каждого временного отсчета рассчитывается среднее значение по всем трассам. Затем это среднее вычитается из каждой трассы. Такой прием уменьшает влияние горизонтальных фоновых отражений.
3. **Компенсация затухания.** Амплитуды более глубоких отсчетов усиливаются линейным коэффициентом, так как георадиолокационный сигнал обычно затухает при распространении в грунте.
4. **Сглаживание и оценка аномальности.** Для подавления случайного шума применяется скользящее среднее. Далее для каждой точки рассчитывается робастная оценка отклонения на основе медианы и MAD:

```text
score = 0.6745 * (abs(x) - median) / MAD
```

Если оценка превышает заданный пользователем порог, точка считается кандидатом в аномалии. После этого близко расположенные кандидаты объединяются путем отбора наиболее сильных локальных максимумов.

## Функционал программы

- загрузка георадиолокационных данных из CSV;
- генерация демонстрационных сигналов;
- сохранение демонстрационного CSV;
- включение и отключение этапов предобработки;
- настройка порога аномальности;
- построение цветовой карты радарограммы;
- выделение найденных аномальных зон;
- просмотр A-скана выбранной трассы;
- сохранение отчета с координатами и оценками аномалий.

## Как описать результат в диссертации

В практической части была разработана программа для автоматизированного определения аномалий георадиолокационных сигналов. Входные данные представлены в виде двумерной матрицы амплитуд, где строки соответствуют отдельным трассам, а столбцы - временным отсчетам. Для повышения устойчивости алгоритма реализована предобработка, включающая удаление фоновой составляющей, компенсацию затухания сигнала с глубиной и сглаживание шумов.

Критерием аномальности выбрано робастное отклонение амплитуды от медианного уровня. Такой подход менее чувствителен к единичным выбросам по сравнению с классическим средним и стандартным отклонением. В результате работы алгоритма формируется список участков, характеризующихся резким усилением или ослаблением отраженного сигнала. Эти участки отображаются на радарограмме и сохраняются в отчете.

Разработанное программное обеспечение может применяться для предварительного анализа георадиолокационных данных при поиске неоднородностей грунта, пустот, металлических объектов, зон повышенной влажности и других объектов, вызывающих заметное изменение формы или амплитуды отраженного сигнала.
