# GPR аномалии

Тема: **Разработка алгоритма и программного обеспечения по определению аномалий георадиолокационных сигналов**.

Приложение предназначено для первичного анализа георадиолокационных B-scan данных при археологической разведке. Программа помогает находить перспективные зоны, которые могут соответствовать погребальным камерам, полостям, нарушенным слоям грунта, каменным конструкциям или другим подповерхностным объектам.

Важно: программа не является окончательным доказательством наличия захоронения. Она формирует предварительную интерпретацию GPR-данных и обоснование, почему участок стоит проверить дополнительно.

## Запуск

```powershell
python gpr_anomaly_app.py
```

Если команда `python` не работает, установите Python 3.10+ и выберите его в Visual Studio Code.

## Запуск REST API

API-модуль реализован в файле `api.py` и использует тот же алгоритм обработки, что и графическое приложение.

Установка дополнительных зависимостей:

```powershell
python -m pip install -r requirements.txt
```

Запуск сервера:

```powershell
uvicorn api:app --reload
```

Основные endpoint'ы:

- `GET /health` - проверка состояния сервиса;
- `POST /analyze` - загрузка `.gpr`, `.gpr2`, `.csv`, `.txt`, `.dat` и анализ аномальных зон;
- `POST /metrics/iou` - расчет IoU для двух бинарных масок.

U-Net-модуль находится в `segmentation_model.py`. Если PyTorch и обученные веса недоступны, API использует совместимый fallback-режим на основе робастного статистического алгоритма, поэтому основное приложение продолжает работать без тяжелых зависимостей.

## Поддерживаемые файлы

Приложение может открывать:

- `.csv` - числовая матрица амплитуд;
- `.gpr` - текстовый или бинарный GPR-файл;
- `.gpr2` - текстовый или бинарный GPR-файл;
- `.txt`, `.dat` - дополнительные текстовые/бинарные варианты.

Так как `.gpr` и `.gpr2` не являются единым открытым стандартом, у разных георадаров структура файла может отличаться. Программа пытается автоматически распознать:

- текстовую матрицу;
- бинарный `int16`;
- бинарный `uint16`;
- бинарный `float32`;
- возможное смещение заголовка;
- количество отсчетов в трассе.

Если конкретный прибор использует закрытый формат, потребуется пример файла и описание структуры. После этого в загрузчик можно добавить точный парсер.

## Логика анализа

Алгоритм состоит из нескольких этапов.

1. **Импорт данных.** Файл преобразуется в матрицу: строки - трассы, столбцы - временные отсчеты/глубинные интервалы.
2. **Dewow-фильтрация.** Удаляется медленная низкочастотная составляющая сигнала.
3. **Удаление фона.** Из каждого отсчета вычитается среднее значение по профилю, что снижает влияние горизонтальных слоев.
4. **Компенсация затухания.** Глубинные отсчеты усиливаются, чтобы слабые глубокие отражения не терялись.
5. **Сглаживание.** Подавляются случайные шумы вдоль трассы и между соседними трассами.
6. **Робастная нормализация.** Используются медиана и MAD, чтобы алгоритм был устойчив к выбросам.
7. **Расчет score.** Для каждой точки рассчитывается оценка аномальности.
8. **Группировка.** Близкие точки объединяются в связные зоны интереса.
9. **Обоснование.** Для каждой зоны рассчитываются размер, пик score, уверенность и текстовое объяснение.

## Интерпретация для археологии

Зона может считаться перспективной, если:

- она является связной и компактной;
- расположена не только в поверхностном шумовом слое;
- имеет высокий контраст относительно фона;
- повторяется на соседних профилях;
- совпадает с археологическим контекстом местности;
- подтверждается несколькими проходами сканирования.

Перед раскопками рекомендуется проводить дополнительную проверку:

- повторный GPR-профиль в перпендикулярном направлении;
- привязка координат GPS/ГНСС;
- сравнение с картой рельефа;
- экспертная интерпретация радарограмм;
- контрольные шурфы только после комплексной оценки.

## Практический результат

Программа стала пригодной для первичного полевого анализа:

- загружает `.gpr`, `.gpr2`, `.csv`;
- работает с чистыми и шумными файлами;
- ищет не отдельные точки, а зоны интереса;
- выделяет зоны на радарограмме;
- позволяет увеличивать и перемещать карту;
- показывает A-scan выбранной трассы;
- формирует отчет с объяснением результата.
